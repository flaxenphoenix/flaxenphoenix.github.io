Permutation Swaps and Hamming Distance

As part of my project, I had to calculate a variant of the Hamming distance between two permutation strings.

The calculation of the Hamming distance between two binary strings can be easily calculated, such as with the following Java code.

  int hammingDistance = 0;
  char[] string1, string2;
  for (int i = 0; i < s1.length; i++) {
    hammingDistance = s1[i] != s2[i] ? hammingDistance +1 : hammingDistance;
  }
  return hammingDistance;

Note, the binary strings are the character arrays here

However, with a permutation string, the distance is not the difference between each individual element within the string but instead the minimum number of swaps between two elements within the string (adjacency is not a requirement), and as such a function to calculate this distance must be capable of permuting the string and counting the number of swaps without affecting the original string. The code that I wrote to do this is as follows:

public int R(int[] a, int[] b) {
		// Mapping
		int[] C = new int[a.length];
		HashMap map = new HashMap<>();
		for (int i = 0; i < a.length; i++) {
			map.put(a[i], i);
		} // Do not join these loops
		for (int i = 0; i < b.length; i++) {
			C[i] = map.get(b[i]);
		}

		// Counting
		int loopCount = 0;
		ArrayList past = new ArrayList();
		boolean newCycle = true;
		for (int i = 0; i < C.length;) {
			if (past.contains(C[i])) {
				newCycle = true;
				i++;
			} else {
				if (newCycle) {
					newCycle = false;
					loopCount++;
				}
				past.add(C[i]);
				i = C[i];
			}
		}
		return a.length - loopCount;
	}
